## 📝 결론

### 오늘 회의 요약

---

### 1. 백엔드 (Backend)

- **로그인**
  - 로그인 화면에서 API 연동
  - 최소 기능으로 토큰 추가
  - 토큰 저장 필요
- **홈아이디**
  - `userId` 파라미터로 요청 시 채널 ID 및 이름 리스트 반환
  - 배열을 반복문으로 처리하여 프론트엔드에 전달
  - 수정 및 삭제 API는 아직 구현되지 않음
- **경로 추가 화면**
  - 경로 설정 등 모든 기능을 폼에서 처리
  - 폼에서 데이터를 반환하도록 구현 예정
- **손녀 기준 화면**
  - 채널 ID와 유저 ID를 통해 모든 데이터를 가져와 프론트엔드에서 렌더링
  - 최적화 방안은 우선 구현 후 검토 예정

---

### 2. z-index 관리

- **z-index 체계 확립 필요성 강조**
  - **Toss 스타일 적용 사례**
  - z-index를 1000 단위로 분류
  - `0`번: 메인 화면
  - `1000`번대: 레이아웃
  - `4000`번대: 캔버스
  - `6000`번대 이상: 모달, 알림(alert) 등 오버레이 요소
- **사용 시기**
  - Absolute 포지셔닝은 꼭 필요한 경우에만 제한적으로 사용
- **추가 논의 사항**
  - 레이아웃 헤더 높이에 대한 결정은 @Zen의 테스트 결과를 바탕으로 추후 공유 예정

---

### 3. GitHub Actions 관련 불편 사항

- **수정 요청 프로세스**
  - GitHub Actions에서 '확인' 라벨 자동 부착
- **CI 테스트 이슈**
  - 조건부 실행 방법 모색 필요 (현재 메일이 지속적으로 발송됨)
- **Reviewer 설정 문제**
  - 조건부 설정 불가, 항상 2명의 리뷰어 필요
  - 수정 후 확인 요청 시 리뷰어가 다시 할당됨
- **자동 머지 불편**
  - 자동 머지 기능 미지원으로 인한 불편함
  - 머지 완료 후 작업 완료로 변경 시 확인 요청 및 리뷰 요청이 반복 발생
- **PR 제목 규칙**
  - 제목에 `#`가 없으면 머지 불가
  - PR 명 확인 작업 옵션 추가 고려
- **프로젝트 연동**
  - 프로젝트 미첨부 시 머지 불가하도록 설정 필요
